# review-board-chatbot — 项目约定

## 代码注释规范

1. 支持且推荐使用中文代码注释。
2. 不要写「你在做什么」,而要在注释里写下「你为什么这么做」。
3. 让类名 / 方法名更好地表达意图,而不是加上一堆解释用的注释。

### 特别写给 AI / Agents / LLMs

1. 不要在代码注释里保留历史的实现版本。
2. 不要在代码注释里写下你对用户提出问题的回答。
3. 不要生成大段晦涩的英文注释,一律使用中文注释;但不要去翻译现有功能的英文关键词(如 token、fortune、signature、webhook、HMAC、SeaTalk、OpenAPI 等)。
4. 不要修改或删除与当前任务不相关的现存注释,特别是那些解释「为什么这么做」、记录待办(TODO)和现存问题的注释。
